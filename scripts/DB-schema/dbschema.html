Database Schema (Developer Reference)

This page documents the current application database schema as defined by the SQL scripts in scripts/ and the setup/migration entrypoints.

Source of Truth and Drift

  1. Baseline schema: scripts/create-postgres-tables.sql
  2. Migrations/additions: scripts/add-*.sql, scripts/create-*-table.sql, scripts/migrate-*.sql, scripts/fix-*.sql
  3. RLS policies: scripts/enable-rls.sql

The setup API at app/api/postgres/setup/route.ts includes an older schema and does not cover newer tables/columns. Database dumps include legacy billing tables (see “Legacy/operational tables” below).

Entity Relationships (High Level)

  • users 1--* teams (owner)
  • users 1--* team_members
  • teams 1--* pathways
  • pathways 1--* phone_numbers (legacy) and 1--1 phone_numbers (current)
  • pathways *--* phone_numbers via pathway_phone_numbers
  • users 1--* phone_numbers
  • users 1--* calls
  • users 1--* call_logs
  • users 1--* payments
  • users 1--* admin_audit_logs
  • users 1--1 wallets (legacy)
  • wallets 1--* wallet_transactions (legacy)

Row Level Security (RLS)

Source: scripts/enable-rls.sql

RLS is enabled on:

  • users, phone_numbers, teams, team_members
  • pathways, activities, invitations

Policies use current_user_id() which reads app.current_user_id from the session setting.

Legacy/Operational Tables (from development_export.sql)

These exist in database dumps and are referenced by billing code, but are not created by the current scripts.

wallets legacy

  • id BIGINT PK (sequence)
  • user_id UUID NOT NULL
  • balance_cents BIGINT DEFAULT 0
  • updated_at TIMESTAMPTZ DEFAULT now()

wallet_transactions legacy

  • id BIGINT PK (sequence)
  • wallet_id BIGINT NOT NULL
  • amount_cents BIGINT NOT NULL
  • type TEXT NOT NULL
  • provider_txn_id TEXT NULL
  • metadata JSONB NULL
  • created_at TIMESTAMPTZ DEFAULT now()
  • gateway TEXT NULL (added via migration)

number_subscriptions legacy

  • id BIGINT PK (sequence)
  • user_id UUID NOT NULL
  • phone_number VARCHAR NOT NULL
  • paypal_sub_id TEXT NOT NULL
  • status TEXT NOT NULL
  • next_billing_at TIMESTAMPTZ NOT NULL
  • created_at TIMESTAMPTZ DEFAULT now()

activities and invitations setup API only

These are defined in app/api/postgres/setup/route.ts and exist in dumps, but are not present in scripts/create-postgres-tables.sql.
